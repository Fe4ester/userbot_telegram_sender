# tg-spam

Локальное приложение для рассылки в Telegram-группы через юзербота с UI и CLI.

Правило отправки: сообщение уходит только в группы, где юзербот админ/владелец.  
Если группа не существует, нет доступа или нет админ-прав, цель пропускается и пишется в лог.

## Установка

```bash
poetry install
```

## Запуск UI (в браузере)

```bash
poetry run tg-ui --host 127.0.0.1 --port 8787
```

Открой `http://127.0.0.1:8787`.

Или одной командой с авто-установкой зависимостей:

```bash
./run-ui.sh
```

## Запуск UI (встроенное desktop окно)

```bash
poetry run tg-ui-desktop --host 127.0.0.1 --port 8787
```

Это тот же локальный веб-интерфейс, но в окне приложения.

Быстрый запуск с авто-установкой зависимостей:

```bash
./run-desktop.sh
```

## Что есть в UI

- `Settings`: текст рассылки по выбранному аккаунту, `parse_mode` (`HTML`/`Markdown`), предпросмотр.
- `Userbot`: мультиаккаунты (добавить/удалить/переключить), `api_id`, `api_hash`, `session`, авторизация через телефон, код и 2FA.
- `Groups`: динамическое добавление/удаление групп и быстрый импорт групп из истории аккаунта Telegram.
- `Logs`: просмотр логов с фильтром `DEBUG/INFO/WARNING/ERROR` и пагинацией.
- `Status`: кнопка проверки статуса аккаунта (авторизация, premium, количество групп, эвристика по спамблоку через `@SpamBot`).
- `Start/Stop`: запуск рассылки и аккуратная остановка.
- Бесконечная отправка через `repeat=0`.

Настройки сохраняются в `settings.yml`.

## Как тестировать безопасно

1. Сохрани настройки.
2. Нажми `Запустить`.
3. Проверь вкладку `Логи` и файл `logs/broadcast.log`.
4. Убедись, что невалидные/недоступные группы уходят в `skipped`.

`repeat=0` в карточке группы означает отправку без лимита, пока не нажмешь `Остановить`.  
Текст и список групп теперь независимы для каждого аккаунта.

## Вход в аккаунт через UI

1. Во вкладке `Юзербот` укажи `api_id`, `api_hash`, `session`.
2. Нажми `Получить код`, введи номер телефона в формате `+7...`.
3. Введи код из Telegram и нажми `Подтвердить код`.
4. Если аккаунт с 2FA, введи пароль и нажми `Подтвердить 2FA`.
5. Нажми `Проверить вход`.

## CLI режим (если нужен)

```bash
poetry run tg-broadcast --config broadcast.yml
```

## Сохранение состояния

Состояние хранится в системной папке приложения (не в директории проекта):

- Windows: `%APPDATA%\TG Broadcaster`
- Linux/macOS: `~/.tg_broadcaster` (или `$XDG_CONFIG_HOME/tg_broadcaster`)

Там сохраняются:

- `settings.yml` (аккаунты, группы, тексты)
- `sessions/` (Telethon-сессии)
- `logs/` (логи)

Поэтому после перезапуска не нужно заново добавлять группы/текст и повторно логиниться.

## Сборка

### macOS бинарник

```bash
./build-macos.sh
```

Результат: `dist/TG-Broadcaster` и `dist/TG-Broadcaster.app`.

### Windows EXE

Запусти на Windows:

```bat
poetry lock
poetry install --with dev
build-windows.bat
```

Результат: `dist\TG-Broadcaster\TG-Broadcaster.exe`.

Windows-сборка использует локальный launcher без `pywebview/pythonnet`: сервер стартует внутри `exe`, UI открывается в отдельном app-окне (Edge/Chrome, режим `--app`).
Запуск: двойной клик по `dist\TG-Broadcaster\TG-Broadcaster.exe`.

Если не стартует:

- проверь `%APPDATA%\TG Broadcaster\crash.log`
- проверь `%APPDATA%\TG Broadcaster\launcher.log`
- собери debug-версию: `build-windows-debug.bat`
- запусти `dist\TG-Broadcaster-Debug\TG-Broadcaster-Debug.exe` и посмотри текст ошибки в окне
